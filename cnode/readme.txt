===================
收藏话题模块
	1. 单击收藏按钮，发送请求
		前台页面使用JQuery发送Ajax请求

	2. 获取当前话题的_id，获取当前登录的用户信息
	3. 存入到集合中
		话题集合 cnode_topics 里面 创建 字段 collect 数组 包含了所有的收藏该该话题的用户的_id
		用户集合 cnode_users 里面 创建 collectTopic 字段 数组，存放该用户所有的已经收藏的话题的_id

删除话题：
	1. 前台删除话题--按钮(管理员和话题作者本身)
		判断当前登录的账户是管理员还是话题作者本身

	2. 删除话题：
		a. 在话题集合中，删除该条数据
		b. 回复--肿么办？
		c. 在发表该话题的用户的topic数组删除对应的_id
		d. 删除话题，减去金币!

用户权限：
	管理员  level 1 
	普通账户 level 0

回复点赞：
	1. 用户必须登录
	2. 点赞：
		在该话题回复的集合中创建 like 数组 存储所有已经点赞该回复
		点赞的是哪个回复 
		在like数组中存储用户的_id

	3. 页面位置 判断当前登录的用户的_id是否在like的数组中
	4. 已点赞，样式为红色，否则为黑色
	5. 判断是点赞还是取消点赞

敏感词：
	过滤敏感词

	str.replace 字符串的替换

	
后台管理：
	涉及用户权限：
		只有管理员才能进行登录

	1. 用户管理模块
		a. 查询用户(所有的用户，按条件进行查询)
		b. 修改用户信息（提权--设置该用户为管理员---管理员多个级别）
		c. 禁用用户（禁止用户登录--需要设置字段）
		d. 删除功能

	2. 话题管理模块
		a. 查看话题列表
		b. 对话题进行管理（修改！删除、拉黑、加精、置顶）

	3. 回复管理模块
		同上！

	4. 分类管理 
		查看、添加、修改
		删除(删除该分类、删除该分类下所有的话题、关联对应的用户、关联对应的回复)

	5. 友情链接管理
		查看
		添加 后台添加的友情链接，默认是审核通过
		删除
		拉黑
		编辑
